# 命令システム概要

## 目的

本書はゲーム内で使用するAIプログラムおよび命令システム全体の設計方針を定義する。

個々の命令の詳細仕様は各カテゴリの仕様書に記載し、本書では命令システム全体の共通仕様のみを扱う。

---

# 基本理念

本ゲームのAIは、プレイヤーが作成したプログラムを実行してロボットを自律制御する。

プログラムはリアルタイムで繰り返し実行され、現在の戦況に応じてロボットの行動を決定する。

プレイヤーは戦闘中に直接ロボットを操作することはできない。

---

# プログラムモデル

プログラムはフローチャートで構成される。

各命令はノードとして表現され、ノード同士を接続することで実行順序を定義する。

プログラムは開始ノードから実行され、接続に従って次の命令へ遷移する。

各Instruction Definitionは、命令が選択できる出力パスを定義する。

Nodeは出力パスIDごとの接続先Node IDを`connections`として保持する。Instructionは実行結果として選択した出力パスIDを返し、AI Engineは対応する接続先Nodeへ遷移する。接続自体は条件を評価しない。

接続は制御フローだけを表し、値の受け渡しには使用しない。命令間の値の受け渡しにはレジスタまたはメモリを使用する。

---

# 実行モデル

AIはゲーム内の一定周期（Tick）ごとに実行される。

各Tickでは以下の順序で処理される。

1. センサー情報取得
2. プログラム実行開始
3. 命令実行
4. CPU制限到達または終了命令で停止
5. 行動結果をシミュレーターへ送信

プログラムは戦闘開始時に開始ノードから実行される。CPU上限でAI実行を終了した場合、次Tickは保存された次実行ノードから再開する。

レジスタ、フラグ、コールスタック、永続AIメモリはTickをまたいで保持される。

---

# 命令の構成

すべての命令は以下の要素で構成される。

* Instruction ID
* パラメータ定義
* 出力パス定義
* 実行時の副作用
* CPU消費量

命令の見た目や編集方法はエディタが担当し、命令システムは動作のみを定義する。

---

# 命令カテゴリ

命令は以下のカテゴリに分類する。

## 制御命令

プログラムの進行を制御する。

例

* 開始
* 終了
* 待機

---

## 分岐命令

条件に応じて実行経路を変更する。

例

* IF
* 比較
* ジャンプ
* CALL
* RETURN

---

## センサー命令

センサー情報を取得する。

例

* 敵の有無
* 距離
* 方位
* 被弾状態

---

## 演算命令

数値演算を行う。

例

* 加算
* 減算
* 乗算
* 除算
* 比較

---

## メモリ命令

内部状態を保存・取得する。

例

* 代入
* 読込
* 初期化

---

## 行動命令

ロボットの行動を指示する。

例

* 移動
* 旋回
* 武器使用
* ロックオン

---

## 特殊命令

その他の特殊処理を行う。

例

* 乱数取得
* デバッグ
* 将来追加命令

# 命令の実行

1つの命令は1回の実行で完結する。

命令は現在のゲーム状態を参照し、必要に応じて内部状態を変更する。

命令自身は時間経過を持たない。

継続動作はAIが毎Tick繰り返し実行されることで実現する。

---

# CPU制限

1Tickで実行できる命令数には上限を設ける。

CPU上限に到達した時点で、そのTickのAI実行は終了する。

CPU上限によって終了した場合も、生成済みの行動要求はそのTickのシミュレーションで使用する。

次のTickのAI実行は、AI Runtime Stateに保存された次実行ノードから再開する。終了ノードへ到達した場合のみ、次TickはProgramの開始ノードから実行する。

条件分岐を含む循環は、Tick開始時に更新されるセンサー値やAI Runtime Stateの変化によって脱出できる可能性があるため、構造エラーとしない。

CPU制限の値はゲームバランスに応じて変更可能とする。

各命令のCPU消費量は命令仕様書で定義する。

---

# レジスタとメモリ

AIは計算結果や状態保持のためにレジスタおよびメモリを利用できる。

レジスタ・メモリの個数と型はAI実行エンジンの仕様で定義する。どちらもAI Runtime Stateの一部としてTickをまたいで保持される。

命令はこれらを介して情報を受け渡す。

---

# データ型

命令で扱う値は共通のデータ型を使用する。

初期バージョンでは以下を想定する。

* 整数
* 真偽値
* 方位
* 距離
* オブジェクト参照

数値は共通データ規約に従い、符号付き32bit整数とする。小数は使用しない。

新しい型を追加できる設計とする。

---

# エラー

命令そのものは構文エラーを持たない。

構造上の問題は Program Validator が検出する。

AI実行中に異常な入力が発生した場合は、各命令が安全な既定値を返すことで処理を継続する。

ゲーム全体を停止させる例外は発生させない。

---

# 拡張性

新しい命令は既存命令を変更せず追加できることを原則とする。

追加時に必要となる情報は以下とする。

* 命令ID
* 表示名
* カテゴリ
* パラメータ定義
* 出力パス定義
* CPU消費量
* 実行処理

既存カテゴリに属さない場合は、新カテゴリを追加できる。

---
