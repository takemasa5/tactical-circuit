# AIプログラミング共通概念

## 目的

本書はAIプログラム全体で共通して使用する概念および用語を定義する。

各命令仕様書は本書の定義を前提とし、同一内容を重複して記述しない。

本書では実装方法ではなく、概念のみを定義する。

---

# AI

AIとは、ロボットを制御するプログラム全体を指す。

AIはプレイヤーが作成する。

戦闘中にAIは一定周期で繰り返し実行される。

AI自身は画面表示や物理演算を行わない。

---

# プログラム

プログラムとは、命令ノードを接続したフローチャート全体を指す。

プログラムは開始ノードを起点として実行される。

実行順序はノード間の接続によって決定される。

---

# ノード

ノードとは、プログラムを構成する最小単位である。

1つのノードは1つの命令を表す。

ノードは以下の情報を持つ。

* 命令種別
* パラメータ
* 入力端子
* 出力端子

ノードはプログラム編集時のみ変更できる。

---

# 接続

接続とは、ノード間の実行順序を定義するリンクである。

接続先が次に実行される命令となる。

分岐命令は複数の接続を持つことができる。

---

# Tick

Tickとはゲーム内部時間の最小更新単位である。

ゲーム内のすべての処理はTick単位で進行する。

AIもTickごとに実行される。

Tickの長さはゲーム全体で共通とする。

---

# AI実行

AI実行とは、1Tick内でプログラムを評価する処理を指す。

AI実行では以下を行う。

* センサー情報取得
* 命令実行
* 行動決定

AI実行の結果はシミュレーターへ渡される。

---

# CPU

CPUとは、1Tick中に実行できる命令量を表す論理的な制限である。

CPUは現実のCPU性能とは無関係であり、ゲームバランスのための概念である。

各命令はCPUを消費する。

CPU上限に到達すると、そのTickのAI実行は終了する。

CPU量や消費値はAI実行エンジンで定義する。

---

# 命令

命令とは、AIが実行する処理単位である。

命令は入力を受け取り、結果を出力し、必要に応じて内部状態やロボットへの指示を更新する。

命令はカテゴリごとに分類される。

---

# パラメータ

パラメータとは、命令ごとに保持する設定値である。

例

* 定数
* 比較演算子
* 対象レジスタ
* 武器番号

パラメータはプログラム編集時に設定し、戦闘中には変更されない（命令仕様で明示される場合を除く）。

---

# レジスタ

レジスタとは、AIが一時的な計算結果を保持するための高速な作業領域である。

主な用途は以下とする。

* センサー値の保持
* 演算結果の保持
* 命令間の値受け渡し

レジスタ構成はAI実行エンジンが定義する。

---

# メモリ

メモリとは、Tickをまたいで保持されるAI内部状態である。

主な用途は以下とする。

* 状態管理
* カウンタ
* フラグ
* タイマー

メモリはレジスタより長い寿命を持つ。

---

# フラグ

フラグとは、真偽値によって状態を表現するデータである。

命令の結果やAI内部状態を保持する用途で使用する。

実装方法はAI実行エンジンで定義する。

---

# センサー情報

センサー情報とは、ロボットが外界から取得できる情報である。

例

* 敵の存在
* 距離
* 方位
* 被弾
* 障害物

AIはセンサー情報のみを通して外界を認識する。

---

# 行動命令

行動命令とは、ロボットへ行動要求を送る命令である。

例

* 移動
* 旋回
* 発射
* ロックオン

行動命令は即座に物理演算を行わず、シミュレーターへ要求を送る。

---

# 行動要求

行動要求とは、AIがシミュレーターへ送信する指示である。

シミュレーターは受け取った要求を次の更新処理で実行する。

AIはロボットの位置や速度を直接変更しない。

---

# ワールド状態

ワールド状態とは、ゲーム世界全体の現在状態である。

以下を含む。

* ロボット
* 弾
* エフェクト
* マップ
* 時間

AIはワールド状態を直接変更してはならない。

---

# Deterministic（決定論）

ゲームは決定論的シミュレーションを基本とする。

同じ初期状態・同じ入力・同じ乱数シードで実行した場合、常に同一結果を得られることを目標とする。

これによりリプレイ、デバッグ、AI比較を容易にする。

---

# エラー

AI実行中の異常入力によってゲーム全体を停止させてはならない。

命令は安全な既定値を返し、AI実行を継続する。

構文エラーや接続エラーは Program Validator が事前に検出する。

---
# Execution Context

## 概要

Execution Context（実行コンテキスト）とは、1TickのAI実行中のみ存在する実行環境である。

AIが命令を実行する際に必要な情報を一元管理し、命令間で共有する。

Execution Contextはゲーム世界そのものではなく、AIがゲーム世界を解釈・操作するための実行環境である。

AIのすべての命令は Execution Context を介して情報を取得し、結果を書き込む。

---

# 目的

Execution Context を導入する目的は以下とする。

* 命令の責務を統一する
* AI Engine と Simulator を分離する
* 命令が World State を直接変更しないようにする
* テストしやすい構造を実現する
* 将来の命令追加を容易にする

---

# ライフサイクル

Execution Context は AI実行開始時に生成される。

AI実行終了時に破棄される。

次のTickでは新しい Execution Context が生成される。

Tickをまたいで保持される状態は保持しない。

継続状態はメモリ領域またはRobot状態へ保存する。

---

# Execution Context が保持する情報

Execution Context は以下の情報を保持する。

## 実行情報

* Tick番号
* AI実行回数
* Program Counter
* CPU使用量
* CPU残量

---

## 実行対象

* 実行中ロボット
* 実行中プログラム

---

## AI状態

* レジスタ
* フラグ
* 一時変数
* コールスタック
* ジャンプ情報

---

## センサー情報

AI開始時点で取得したセンサー情報を保持する。

例

* 敵情報
* 距離
* 方位
* 被弾状態
* 自機状態

AI実行中はこのスナップショットを参照する。

同一Tick中にセンサー値が変化することはない。

---

## 行動要求

AIが決定した行動はExecution Contextへ蓄積する。

例

* 移動要求
* 旋回要求
* 武器発射要求
* ロックオン要求

命令は直接ロボットを動かさない。

AI終了後にシミュレーターが行動要求を取得し、ゲーム世界へ反映する。

---

# World Stateとの関係

Execution Context は World State を読み取ることができる。

ただし World State を直接変更してはならない。

ゲーム世界を変更する必要がある場合は行動要求を生成する。

```
World State

↓

Execution Context

↓

Action Queue

↓

Simulator

↓

World State
```

---

# 命令との関係

すべての命令は

入力

↓

Execution Context

↓

処理

↓

Execution Context更新

という共通モデルで実行される。

命令は

* World State
* Robot
* Simulator

を直接操作しない。

---

# センサー取得

センサーはAI開始時に一括取得する。

Execution Context はその結果を保持する。

同一Tickではすべての命令が同じセンサー情報を参照する。

これにより実行順序による差異を防ぐ。

---

# 行動要求

行動命令はロボットを直接操作しない。

Execution Context に要求を書き込む。

AI終了後、Simulator が要求を読み取り実行する。

例

```
MOVE命令

↓

Execution Context

moveForward = true

↓

Simulator

↓

移動処理
```

---

# レジスタとメモリ

Execution Context はレジスタへアクセスできる。

またRobotに属する永続メモリにもアクセスできる。

役割は以下の通りとする。

| 種類          | 寿命       |
| ----------- | -------- |
| レジスタ        | AI実行中のみ  |
| メモリ         | Tickをまたぐ |
| World State | ゲーム全体    |

---

# CPU管理

CPU使用量はExecution Contextが管理する。

命令実行ごとにCPU消費量を加算する。

CPU上限へ達した場合、そのTickのAI実行を終了する。

---

# デバッグ

Execution Context はデバッグ情報の取得元となる。

例

* 現在命令
* 実行履歴
* CPU使用量
* レジスタ状態
* メモリ状態
* センサー値
* 行動要求

リプレイやデバッグ画面はExecution Contextを参照して表示を行う。

---

# 将来拡張

今後追加される概念についても、本書に定義した上で各仕様書から参照する。

新しい用語を導入する場合は、各命令仕様へ直接記載せず、本書へ追加すること。

これにより、AI仕様全体で用語の意味を統一する。

---

## Execution Context

Execution Context は将来的に以下の情報を保持できる。

* 命令プロファイラ
* 実行時間
* ブレークポイント
* トレースログ
* デバッグ変数
* AIバージョン情報

既存命令を変更せず拡張可能な構造とする。
