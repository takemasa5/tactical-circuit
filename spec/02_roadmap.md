# 開発ロードマップ

## 目的

本書はプロジェクトの実装順序を定義する。

各フェーズは独立して完了可能であり、各フェーズ終了時には動作確認可能な成果物を得ることを目的とする。

実装は可能な限り小さな単位で完了させ、動作確認を繰り返しながら進める。

---

# 開発方針

各フェーズは以下を満たすこと。

* ビルド可能
* 実行可能
* 自動テストが成功する
* 次フェーズへ影響を与えない

途中のフェーズでもゲーム全体が動作することを目標とする。

# Phase 3

## Program Validator

### 目的

プログラムの静的解析を行い、AI実行前に構造上の問題を検出する。

AI Execution Engine は、Validator を通過したプログラムのみを受け付ける。

### 実装

* 開始ノードの存在確認
* 未接続ノード検出
* 到達不能ノード検出
* 不正な接続検出
* 必須パラメータ検証
* 命令ごとの入力値検証
* ラベル重複検出
* 存在しないジャンプ先検出
* 純粋な循環プログラムの解析
* Warning / Error の分類
* エラー位置の取得

### 完了条件

* プログラム全体を静的解析できる
* エラー一覧を返却できる
* Warning一覧を返却できる
* 不正なプログラムは AI Engine に渡されない
* 純粋な循環プログラムはWarningとなり、AI Engineでの実行を妨げない

# 開発ルール

* 各フェーズ開始前に関連仕様書を確認すること。
* 仕様変更時は、実装より先に仕様書を更新すること。
* 実装と仕様書に差異がある場合は仕様書を正とする。
* Program Validator を通過していないプログラムを AI Execution Engine に渡してはならない。
* 各フェーズ完了時には、自動テストおよび動作確認を実施すること。
