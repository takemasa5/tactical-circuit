# 2026-06-26
1点だけ、将来を見据えた改善提案

この仕様書を読んでいて、私は 「Node」と「Instruction」を明確に分離する ことも将来的に検討する価値があると考えています。

現在の仕様では「1つのノードは1つの命令を表す」としていますが、論理的には次のような関係です。

Program
 ├─ Node
 │    ├─ Position
 │    ├─ Connections
 │    └─ Instruction
 │          ├─ Type
 │          ├─ Parameters
 │          └─ Metadata

つまり、

Instruction は「何をするか」（意味）
Node は「どこに置かれ、どこへつながるか」（グラフ上の表現）

という分離です。
