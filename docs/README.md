# ドキュメント案内

| 場所             | 内容                                          | 実装・レビューでの扱い                              |
| ---------------- | --------------------------------------------- | --------------------------------------------------- |
| `product/`       | プロダクトの目的、共通概念                    | 仕様の背景として参照する                            |
| `specs/current/` | 現在の実装が満たすべき仕様                    | 通常のレビュー基準にする                            |
| `specs/planned/` | POと設計者が定めた将来仕様                    | Issueが指定した範囲以外は実装・レビュー対象にしない |
| `planning/`      | マイルストーン、Issue雛形、申し送り、設計メモ | 作業の計画と追跡に使う                              |
| `decisions/`     | POが承認した重要な設計判断                    | 必要になった時点で追加する                          |

仕様を実装するときは、選択したIssueのAcceptance Criteriaを満たす範囲だけを`specs/planned/`から実装し、実装した内容を`specs/current/`へ反映する。
