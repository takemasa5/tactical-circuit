# coordinate_system.md

# 座標系

## 目的

本書はゲーム内の座標系など共通の概念を定義する。

---

# 基本理念

座標系や角度はすべてのサブシステムで共通の概念として扱う。

---

## 座標系

左下を(0, 0)とし、右方向をX成分、上方向をY成分とする。
X成分、Y成分ともに論理的な単位とし、ピクセル等の物理量には依存しないものとする。
小数点以下は持たず、常に整数とする。

位置は`Position`、ベクトルは`Vector`、大きさは`Size`として以下の共通構造で表現する。

```ts
type Position = {
  x: number;
  y: number;
};

type Vector = {
  x: number;
  y: number;
};

type Size = {
  width: number;
  height: number;
};
```

各フィールドは符号付き32bit整数とする。

Robot、弾、障害物の当たり判定は、`Position`と`Size`で表す軸平行矩形を使用する。弾の進行方向と1Tickあたりの移動量は`Vector`で保持する。

---

## 角度

ロボットは向きを持つ。
角度は度数法の整数で表現する。
角度は正面をゼロ度とし、角度の値は時計回りに増加する。
角度はマイナス値も許容し、-90は270と同義として扱う。
角度は360以上の値も許容する。450は90と同義として扱う。
JSONへの保存時は0以上360未満の範囲へ正規化する。

---

## Tick

ゲーム内時間の単位をTickとする。
一時停止やスロー再生ができるようにTickは実時間とは無関係なものとする。
衝突判定のすり抜けを防止するため、1Tickは十分に小さい時間とする。
通常シミュレーション開始時のTickは0とし、通常シミュレーションが1回完了した場合のみ1増加する。

---

## 速度

速度は1Tickあたりの移動距離とする。
