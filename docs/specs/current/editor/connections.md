# 接続編集仕様

> Status: Draft

## 目的

本書はProgram EditorにおけるNode間の接続作成、変更、削除を定義する。

---

# 接続の識別

接続は接続元Node IDと出力パスIDの組み合わせで識別する。

独立したConnection IDは持たない。接続先Node IDは接続の識別子に含めない。

---

# 接続作成

接続作成は以下を入力とする。

- 接続元Node ID
- 出力パスID
- 接続先Node ID
- 操作日時

次のすべてを満たす場合だけ接続を作成する。

- 接続元Nodeと接続先NodeがProgram内に存在する
- 出力パスIDが接続元NodeのInstruction Definitionに存在する

`enabled`が`false`のInstruction Definitionを参照する既存Nodeも編集できる。`enabled`は新しいNodeの作成候補だけに影響する。

自己への接続を許容する。複数のNodeから同じNodeへの接続も許容する。

対象出力パスが未接続の場合は接続を追加する。すでに別のNodeへ接続されている場合は、同じ操作で接続先を置き換える。

すでに同じNodeへ接続されている場合はProgramを変更しない。

循環、到達可能性、requiredな出力パスなどProgram全体の妥当性は判定しない。

---

# 接続削除

接続削除は接続元Node ID、出力パスID、操作日時を入力とする。

接続元Nodeが存在し、対象出力パスに接続が存在する場合、そのプロパティを`connections`から削除する。

接続元Nodeが存在しない場合は失敗とする。対象出力パスが未接続の場合はProgramを変更しない。

requiredな出力パスの接続も編集途中では削除できる。不足した接続の検出はProgram Validatorが担当する。

---

# UI操作

接続は出力ポートから接続先Nodeの入力領域へドラッグして作成する。

接続のドラッグ中は、ドラッグを開始した出力ポートの右辺中央から現在のポインター位置までプレビュー接続線を表示する。プレビュー表示はProgramを変更しない。

接続先が確定した時点で1回の編集操作とする。接続可能な領域以外でドラッグを終了した場合は変更せず、プレビュー接続線を消去する。

既存接続を選択して削除操作を行うことで切断できる。Node削除に伴う接続削除はNode削除操作の一部とし、独立した履歴項目にしない。

既存接続線の選択領域は表示線より広くし、線の中心から両側8論理ピクセル以上をクリック可能とする。選択領域を広げても接続線の見た目の太さは変更しない。

既存接続線または出力ポートへマウスポインターを重ねた場合、対応するOutput Path Definitionの`description`をツールチップとして表示する。説明表示はProgramを変更しない。

---

# 表示順

Node内の出力ポートはInstruction Definitionの`displayOrder`昇順で表示する。同じ`displayOrder`の場合は出力パスIDの文字列昇順とする。

作成済み接続線の接続元は、対応する出力ポートの右辺中央とする。Node右辺の共通位置へまとめて接続しない。接続先は接続先Nodeの左辺にある入力ポートの中央とする。

各出力ポートの右側には、接続先Nodeの入力ポートと同じ見た目の円形ポートを表示する。接続線の接続元はこの円形ポートの中央とする。

出力ポートは対象出力パスが接続済みの場合に塗りつぶす。入力ポートは1本以上の接続線の接続先になっている場合に塗りつぶす。未接続のポートは塗りつぶさず、接続の不足を外観から判別できるようにする。

接続線およびドラッグ中のプレビュー接続線はNodeより前面に表示し、Nodeの外枠によって隠れないものとする。

Recordのプロパティ順を表示順や実行順として使用しない。
