# クリップボード仕様

> Status: Draft

## 目的

本書はProgram Editorにおけるノード群のコピーと貼り付けを定義する。

---

# クリップボード状態

クリップボードは編集セッション内のアプリケーション専用状態とし、Programへ保存しない。Phase 2ではOSのクリップボードを使用しない。

クリップボードはコピー時点の以下の情報を保持する。

- コピー元Program ID
- コピーしたNode
- Node IDごとの位置
- 存在する場合はNode IDごとのコメント

クリップボード内のNodeはコピー元ProgramのNode IDを維持する。新しいNode IDは貼り付け時に発番する。

---

# コピー

コピーは1つ以上の選択中Nodeを対象とする。

コピー時は対象NodeとEditor専用情報を値として複製し、コピー後に元Programが変更されてもクリップボードの内容が変化しないようにする。

選択Node間の接続だけを保持する。選択範囲外のNodeを接続先とする接続はクリップボードへ含めない。

Parameter ValueがNode参照である場合は以下のように扱う。

- 選択範囲内のNodeへの参照は保持する
- 選択範囲外のNodeへの参照は、そのParameter IDごとクリップボード内の`parameterValues`から除外する

コピー成功時は以前のクリップボードを置き換える。Programと履歴は変更しない。

選択Nodeがない場合はコピーに失敗し、以前のクリップボードを維持する。

---

# 貼り付け

貼り付け先Programの`nextNodeSequence`を使用し、クリップボード内のすべてのNodeへ新しいNode IDを発番する。

発番順はコピー元Node IDの文字列昇順とする。各Nodeを追加するたびに連番を1増加し、最後の値をProgramの`nextNodeSequence`とする。

接続先とNode参照は、コピー元Node IDから新しいNode IDへの対応を使用して置き換える。

貼り付け後の位置はコピー元の位置を維持する。貼り付け位置を自動調整する状態やカウンタは持たない。

貼り付けによってStart命令を参照するNodeが複数になった場合も、`startNodeId`は変更しない。意味上の問題はProgram Validatorが検出する。

---

# 原子性

次の場合は貼り付け操作全体を失敗とし、Programと`nextNodeSequence`を変更しない。

- クリップボードが空である
- 必要なNode IDをすべて発番できない
- クリップボード内で必要な位置情報が欠けている

貼り付け成功時は、すべてのNode追加を1回の編集操作および1回の履歴項目として扱う。

貼り付け後は、新しく追加したすべてのNodeを選択し、それ以前のNode選択と接続選択を解除する。

---

# Programをまたぐ貼り付け

クリップボードはProgramを読み込んだ後も維持し、別のProgramへ貼り付けできる。

選択範囲内の接続とNode参照は新しいNode IDへ置き換えられるため、コピー元ProgramのNode IDには依存しない。
